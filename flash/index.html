WebUSB badge workspace / experimental hardware path

Inspect first. Flash only when every fact checks out.

This guided tool separates a non-destructive BadgeMagic Bluetooth probe from CH582M firmware replacement through the WCH USB bootloader. There is no erase or write on connect; connecting is never permission to erase.

Before the first write

There is no OEM backup or original-firmware restore.

The OEM firmware is read-protected, unavailable, and cannot be backed up or dumped by this page. Replacing it is irreversible unless you already possess a recoverable image for your exact board.

FOSSASIA's open BadgeMagic firmware is a compatible substitute for one exact Micro-USB profile. It is not the original factory firmware.

Step 2 / choose the real device state

Two connections. Two different jobs.

Web Bluetooth cannot install firmware. WebUSB cannot inspect the running BadgeMagic app protocol.

Choose a nearby badge advertising service FEE0. The probe checks for the legacy FEE1 characteristic, disconnects, and changes no content.

Browser detected READ-ONLY FIRST

WCH ISP bootloader via USB

Choose the USB bootloader, not the running Bluetooth badge. The first connection identifies chip 0x82, family 0x16, and configuration without erasing or programming.

Running FOSSASIA or tested FrogAlert firmware: with USB connected, hold KEY2—the physical button nearest the USB connector—for about 2.2 seconds. Release when the dot cue appears, then open the chooser promptly. No RESET or multi-button combo is needed.

OEM, unknown, blank, or broken firmware: the application hook may not exist. Use the guided cold-entry fallback. On a soldered-battery board, electrical isolation is skilled bench work—not an ordinary battery-unplug step.

Physical steps

Normal application KEY2 sequence

  1. 1

    Keep USB connected to a stable data-capable port or phone OTG adapter.

  2. 2

    Press and keep holding KEY2, the button nearest the USB connector.

  3. 3

    Hold for about 2.2 seconds only when running the tested FOSSASIA image or an exact FrogAlert artifact whose KEY2 recovery test passed.

  4. 4

    Release when the dot cue appears, then immediately use Dot or pixel is lit — choose CH582 now. The bootloader window is short.

  5. 5

    Accept only a read-only CH582 result. If the chooser shows nothing, retry once; use the cold-entry guide only if the application path is absent or broken.

Which button should I use?

Badge still advertises over Bluetooth
Use Probe running badge first. It cannot repair firmware.
Badge is blank, frozen, or a flash was interrupted
Use Advanced cold-entry fallback. A soldered battery requires qualified bench isolation.
CH582 is identified
Select and hash-check an exact-board image before confirming any write.
No USB bootloader appears
Retry with a known data cable and direct port. Do not erase based on Bluetooth identity.
What if KEY2 never reaches the bootloader?

Stop before probing or shorting an opened board from a phone. The C3 reset method requires the exact schematic and physical board identification; it is deliberately not an ordinary web-flasher instruction. Review the hardware safety guide and use the CLI path with qualified bench access.

If programming stops

Do not claim success. Keep the exact selected BIN and its hash, disconnect power, repeat the KEY2 recovery sequence, establish a fresh read-only identity, then retry the complete write and verification flow. A replacement USB device must never inherit an active flash session.
